# Release Plan 1.0.0

## Release goal

Ship the first stable standalone broker platform release with a documented broker-sidecar contract, deployable container images, and repeatable local and Kubernetes deployments.

## Exit criteria

- Broker and sidecar source trees live in this repository.
- Broker-sidecar contract ADR is implemented, not just documented.
- Docker Compose smoke test is green from a clean checkout.
- Kubernetes chart deploys a healthy broker + sidecar pod.
- Browser `WebTransport.ready` succeeds against the sidecar endpoint.
- One bidirectional stream open and echo test passes end to end.
- Auth token path and broker bridge path are validated in CI.
- Release pipeline publishes versioned broker and sidecar images.
- Release notes, changelog, and compatibility matrix are generated.

## Workstreams

### 1. Repository extraction

- Move broker code and Maven build into `broker/`.
- Move sidecar code and package metadata into `sidecar/`.
- Move or recreate Dockerfiles per component.

### 2. Contract hardening

- Replace readiness log scraping with admin endpoints.
- Version the contract and enforce it at broker startup.
- Add structured startup metadata to both components.

### 3. Deployment productization

- Finalize `docker-compose.yml` for local and CI.
- Finalize Helm chart defaults for Kubernetes.
- Add example manifests for managed cloud environments.

### 4. Validation

- Add unit tests for config validation.
- Add integration tests for broker launch and companion mode.
- Add browser interop smoke tests.
- Add release-candidate checklist gates in CI.

### 5. Release automation

- Publish broker image `ghcr.io/nextend/broker-platform-broker:1.0.0`.
- Publish sidecar image `ghcr.io/nextend/broker-platform-sidecar:1.0.0`.
- Create Git tag `v1.0.0`.
- Generate changelog and release notes.

## Milestones

| Milestone | Outcome                              |
| --------- | ------------------------------------ |
| `M1`      | Repository scaffold and ADR approved |
| `M2`      | Source extraction complete           |
| `M3`      | Contract endpoints implemented       |
| `M4`      | Compose and Helm deployments green   |
| `M5`      | End-to-end browser interop green     |
| `M6`      | `1.0.0-rc.1` published               |
| `M7`      | `1.0.0` released                     |

## Risks

- Sidecar QUIC/WebTransport behavior may differ across base images and kernels.
- UDP ingress handling varies by cloud and cluster setup.
- Browser-facing certificate handling must stay aligned with broker token hints.

## Non-goals for `1.0.0`

- JNI integration into the broker JVM
- multiple sidecar implementations
- multi-region control-plane automation

## Recommended tag strategy

- `v1.0.0-alpha.N` for extraction and contract work
- `v1.0.0-beta.N` for deployable end-to-end candidates
- `v1.0.0-rc.N` for release candidates
- `v1.0.0` after all exit criteria are met
